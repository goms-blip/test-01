# 가계부 비서 — 가계부와 함께, 생활의 분석

> 가계부앱(`week-5/quest/가계부앱`)이 **데이터 입력**을 담당하고,
> 본 프로젝트는 그 데이터를 보면서 **돈의 흐름 뒤에 있는 생활**을
> 짚어주는 비서 역할을 맡습니다. 분석가의 보고서가 아니라
> 사람의 말투로, 다음 행동을 함께 정해주는 톤이 목표입니다.

```
가계부 앱(입력) → Supabase(저장) → 분석 에이전트(질문/응답) → 사용자
```

## ⚡ 빠른 시작 (오프라인 데모)

서버 없이 브라우저만 있으면 됩니다.

```bash
# 이 폴더로 이동 후
python3 -m http.server 5174
# 또는
npx serve -p 5174 .
```

→ `http://localhost:5174` 접속.

좌측 추천 질문을 누르거나, 입력창에 자유롭게 질문하세요.

### 답변 가능한 질문 11종

1. 총 지출 (이번 달/지난주/오늘/특정 카테고리…)
2. 카테고리별 분포 (% 비중 + 차트)
3. 가장 많이 쓴 날 (전체 또는 카테고리별)
4. 요일별 패턴 (주중 vs 주말 비교)
5. 월말 예측 (현재 속도 기반)
6. 예산 잔여 (`100만원 예산`처럼 발화에서 추출)
7. 절약 조언 (카페·구독·외식·Top1 카테고리 분석)
8. 월간 비교 (이번 달 vs 지난달, 카테고리 변동 Top 5)
9. 잔액 (수입·지출·잔액)
10. 최근 내역 (`최근 5건`)
11. 도움말

자세한 매핑은 `intents.md` 참고.

## 🔌 실제 Supabase 연결

1. `schema.sql` 을 Supabase SQL Editor 에서 실행 → 분석 뷰 추가
2. `mcp.example.json` 을 참고해 Claude Code/Cursor 에 MCP 서버 등록
3. `--read-only` 플래그로 분석 에이전트의 쓰기 차단
4. `SETUP.md` 의 시스템 프롬프트를 에이전트 가이드로 사용

> 자세한 내용은 [`SETUP.md`](./SETUP.md).

## 🧠 설계 메모

- **뷰 우선 전략**: 자주 쓰이는 6가지 분석은 모두 SQL 뷰로 미리 만들어
  에이전트가 호출하는 SQL 을 단순화 (느리고 복잡한 CTE 회피).
- **응답 = 결과 + SQL**: UX 측면에서 "에이전트가 어떤 쿼리를 던졌는지"
  같이 보여주어, 사용자가 결과를 신뢰하고 직접 쿼리도 학습할 수 있게 함.
- **읽기 전용**: 분석 에이전트는 절대 데이터를 변경하지 않음.
  (MCP `--read-only`, RLS 분리 권장 — `schema.sql` 7번 참고)
- **단일 사용자 가정**: 가계부앱이 단일 유저 데모이므로 동일하게 가정.
  멀티유저 확장 시 `user_id` 컬럼 + `auth.uid()` 정책 추가.
