## 2026-04-29 · v0.1 — 초기 구축

**목표**: `prd.md` 기반으로, 가계부앱 데이터를 자연어로 분석하는
분석 에이전트의 데모 + Supabase 연결 자료 일체 준비.

### 1) 결정 사항
- 가계부앱(`week-5/quest/가계부앱`)이 이미 `ledgers` 테이블을
  운영하므로 **스키마 재정의 없이 그대로 사용**, 분석 전용 뷰만 추가.
- 분석 에이전트는 **읽기 전용** 으로 제한
  (`--read-only` MCP 플래그 + read-only 롤 권장).
- 데모는 백엔드 없이 동작해야 OT 발표/스크린샷 촬영이 쉬움 →
  `data.js` 로 인메모리 시드 + 브라우저에서 동작하는 분석 엔진(`agent.js`)
  을 만들고, **Supabase 연결 시 동일 SQL 을 그대로 보내면 되도록** 설계.
- "결과 + 사용한 SQL" 을 함께 노출 — 에이전트의 추론을 투명하게
  보여주는 것이 본 퀘스트의 학습 가치라 판단.

### 4) UX 메모
- 모든 응답은 **텍스트 + 실행 SQL** 두 블록을 항상 같이 노출.
  사용자가 SQL 학습 + 결과 검증 동시에 가능.
- 차트는 분포/요일 패턴 두 핸들러에만 부여 (오버UI 방지).
- 데모는 다크 + fintech 톤. 좌측 사이드바에 누적 통계 + 추천 질문 9개.

### 5) 보안 체크
- `.gitignore` 에 `.env`, `mcp.json` 추가 (anon key 평문 노출 차단).
- `schema.sql` 에 read-only 분석 롤 생성 SQL 주석으로 제공.
- `mcp.example.json` 에는 더미 키만 표기, 실제 키는 사용자가 직접 채움.

### 6) 미해결 / 후속 과제
- **스크린샷 3종**: 데모를 띄워 `이번 달 총 지출은?` →
  `카테고리별 분석해줘` → `100만원 예산 중 남은 금액은? + 절약 추천` 순으로
  캡처해 `screenshot/` 디렉터리에 저장 필요. (PRD 6번 체크박스)
- **실 Supabase 연동 시연**: `SETUP.md` 따라 MCP 등록 후
  Claude Code 에서 `이번 달 카테고리별 지출 SQL 짜서 실행해줘` 같은
  발화로 한 번 동작 확인하면 좋음.
- **멀티유저 확장**: 현재는 단일 유저 가정. 향후 `user_id` 컬럼 +
  `auth.uid()` 정책 + 분석 뷰의 `where user_id = auth.uid()` 조건 필요.
- **자연어 처리 한계**: 라우터는 정규식 기반.
  복잡한 질문(다중 조건/시계열 비교)은 실제 LLM + MCP 조합으로 위임 권장.
