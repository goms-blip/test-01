
## 📝 PRD: 가계부 DB 분석 에이전트 (Account Book Analyzer Agent)

### 1. 프로젝트 개요
* 목적: Supabase에 저장된 가계부 데이터를 에이전트와 연결하여, 자연어 질문만으로 소비 패턴을 분석하고 맞춤형 금융 조언을 제공하는 AI 분석가를 구축합니다.

* 핵심 가치: 데이터 입력(앱)과 데이터 분석(에이전트)을 분리하여, 내 데이터를 자산화하고 AI 비서로 활용하는 경험을 제공합니다.

### 2. 사용자 타겟
* 기존 가계부 앱 퀘스트를 완료하여 Supabase에 10건 이상의 지출 데이터를 보유한 사용자.
* 복잡한 차트나 통계 페이지 대신 대화로 쉽고 빠르게 지출 현황을 파악하고 싶은 사용자.

### 3. 핵심 기능 (User Stories)
### 3.1. DB 연결 및 데이터 조회 (Part 1)
* 기능: Supabase MCP(Model Context Protocol) 또는 API를 통해 가계부 테이블 연동.
* 사용자 시나리오: "에이전트가 내 가계부의 expenses 테이블을 실시간으로 읽어와 답변할 수 있어야 함."

### 3.2. 지출 패턴 분석 (Part 2)
* 기능: 기간별, 카테고리별, 요일별 소비 데이터 가공 및 비교.
* 주요 질문:
* "이번 달 총 지출액은?"
   * "식비 지출이 가장 컸던 날은?"
   * "주말과 주중 중 어디에 돈을 더 많이 쓸까?"

### 3.3. 스마트 절약 조언 및 예측
* 기능: 현재 소비 속도를 바탕으로 한 미래 예측 및 예산 관리 가이드.
* 주요 질문:
* "현재 속도로 쓰면 이번 달 총 얼마를 쓸까?"
   * "50만원 예산 중 남은 금액은?"
   * "불필요한 지출을 줄일 수 있는 항목 추천해줘."

### 4. 기술 스택 및 구조
* Database: Supabase (PostgreSQL)
* Agent: Cursor, Claude, 또는 OpenAI Assistant (MCP 지원 도구 권장)
* Protocol: MCP (Supabase Connector) 또는 직접적인 SQL Query 실행 환경
* Flow: 가계부 앱(입력) → Supabase(저장) → AI 에이전트(분석) → 사용자(결과 확인)

### 5. 단계별 실행 계획 (Roadmap)

### Step 1: 환경 설정
* Supabase 프로젝트 API 키 및 URL 확보.
* 에이전트에 DB 접근 권한 설정 (MCP 설정 파일 업데이트 등).

### Step 2: 기본 분석 수행
* 단순 집계 쿼리 테스트 ("이번 달 얼마 썼어?").
* 에이전트가 테이블 구조(Schema)를 정확히 파악하고 있는지 확인.

### Step 3: 분석 심화 및 고도화
* 복합 질문 테스트 (지난달 vs 이번달 비교).
* 자신만의 독창적인 기능 구현 (예: 소비 등급 부여, 연말 지출 예측 리포트).

### 6. 최종 제출물 체크리스트
* GitHub 저장소: 에이전트 설정 코드 및 SQL 스키마 포함.
* 스크린샷 A (조회): 이번 달 지출액 등 기본 데이터 응답 화면.
* 스크린샷 B (분석): 요일별/카테고리별 패턴 분석 대화 화면.
* 스크린샷 C (조언): 예산 대비 남은 금액 및 절약 제안 화면.
* 가계부 앱 인증: 데이터가 쌓여있는 실제 앱 구동 화면.
