# 🌍 행복을 찾아 떠나는 여행 — Context-Aware AI Travel Agent

> **컨셉**: AI Context(.md) + Supabase DB를 결합해, 일반론이 아닌 *고객님 자신의 행복 데이터*에서 답을 끌어오는 초개인화 여행 에이전트.
>
> **핵심 가치**: "데이터(DB)는 *행동*을 말하고, 컨텍스트(Context)는 *정체성*을 정의한다."

---

## 🎬 데모 시나리오

### 시나리오 1 — PRD 대표 질문
**Q.** "여지껏 다녀온 여행지중에 가장 좋았던 곳은?"

| | Before (컨텍스트 OFF) | After (컨텍스트 + DB) |
|---|---|---|
| 답변 | "일반적으로 동남아나 따뜻한 나라를 추천합니다" | **"홋카이도 (3회 방문, 평균 9.5/10)"** + 9.5+ 순간 5개 인용 |
| 근거 | 통계적 인기 | DB의 실제 행복지수 |

### 시나리오 2 — 제약 검증
**Q.** "동남아 추천해줘"

- **Before**: 태국·베트남·발리·필리핀 일반 추천
- **After**: ⛔ 거절 + *2022-06 다낭 평균 4.8/10* 인용 + "더운 동남아 회피" 컨텍스트 §3-1 인용 + 대안 제시

### 시나리오 3 — 재방문 룰
**Q.** "교토 다시 가도 될까?"

- **Before**: "좋은 여행지예요, 시즌만 맞으면 OK"
- **After**: "⛔ 직전 방문 후 N개월밖에 안 됨. 컨텍스트 §3-3 *3년 룰* 위반"

### 시나리오 4 — 겨울 추천
**Q.** "이번 겨울에 어디 갈까?"

- **Before**: "겨울에는 따뜻한 곳으로 피난 (다낭·푸켓)"
- **After**: "겨울 = 홋카이도. 평균 9.5. 컨텍스트의 *눈·겨울* 취향과 정확히 일치" + 3가지 시나리오

## 🧠 컨텍스트가 만든 차이

| 측면 | Without Context | With Context |
|---|---|---|
| 답변 톤 | 범용 가이드 | 1인칭 비서 |
| 추천 기준 | 통계적 인기 | 사용자의 과거 행복지수 |
| 제약 처리 | 무시 | 4가지 제약을 모두 검증 후 통과한 것만 |
| 데이터 인용 | 없음 | DB row를 직접 인용 (날짜·내용·점수) |
| 부정 답변 | 없음 | "동남아는 추천하지 않습니다" 처럼 *거절*도 함 |

## 🛠️ 기술 스택

- **Frontend**: 단일 `index.html` + Chart.js (CDN)
- **Logic**: Vanilla JS (`agent.js`) — intent router + 데이터 집계
- **Data**: 데모는 `data.js`(오프라인), 실제는 Supabase (PostgreSQL)
- **Context**: `context.md` (LLM 시스템 프롬프트로 직접 주입 가능)

> 💡 실제 LLM (OpenAI / Claude)에 연결할 때는 `context.md` 전체를 system message로,
> Supabase row들을 user message에 결합해 보내면 됩니다. 자세한 건 `SETUP.md` 참고.

1. 🥇 **홋카이도** — 9.45 (24 records, 3 visits)
2. 🥈 **닛코** — 9.0 (3 records, 1 visit)
