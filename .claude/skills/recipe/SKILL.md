# /recipe — 냉장고 재료 기반 레시피 생성

사용자가 `/recipe`를 실행하면, 현재 작업 디렉터리(또는 지정된 경로)의 `ingredients/` 폴더에 있는 JSON 파일들을 모두 읽어 재료 목록을 파악한 뒤, 한 가지 레시피를 제안하고 마크다운 파일로 저장합니다.

## 실행 순서

1. **재료 폴더 탐색**
   - `ingredients/` 폴더를 현재 작업 디렉터리 기준으로 찾습니다.
   - 못 찾으면 하위 디렉터리에서 `ingredients/`를 검색 (`Glob` 사용). 여러 개 발견되면 사용자에게 어떤 폴더를 쓸지 물어봅니다.
   - 폴더가 전혀 없으면 안내 메시지를 출력하고 종료.

2. **재료 JSON 파싱**
   - `ingredients/*.json` 파일을 모두 읽어 아래 필드를 추출합니다.
     - `name` (필수): 재료 이름
     - `quantity` (선택): 수량
     - `category` (선택): 보관 위치 (냉장/냉동/실온)
     - `note` (선택): 특이사항 (유통기한, 신김치 여부 등)
   - 파싱 실패한 파일은 건너뛰되 어떤 파일이 문제인지 사용자에게 알림.

3. **레시피 제안**
   - 다음 조건을 **반드시** 지켜서 레시피 1개를 제안:
     - **1인분 기준**
     - **조리 시간 15분 이내** (준비 시간 포함)
     - **자취생 난이도** — 기본 조리도구(냄비, 후라이팬, 전자레인지)만 사용
     - 가능한 한 **폴더에 있는 재료만** 활용. 소금/후추/설탕/기름 같은 기본 양념은 있다고 가정해도 됨.
   - 재료가 너무 적어서 만들 수 있는 요리가 없다면, 어떤 재료가 추가로 필요한지 안내.
   - `note`에 "신김치", "유통기한 임박" 같은 힌트가 있으면 이를 반영한 레시피를 우선 선택.

4. **마크다운 파일로 저장**
   - 저장 경로: `recipes/{YYYY-MM-DD}_{요리명-슬러그}.md`
     - `recipes/` 폴더가 없으면 만들지 말고, `ingredients/`의 부모 디렉터리를 기준으로 저장 위치를 결정.
   - 파일명 슬러그는 한글은 그대로 두고 공백/특수문자는 `-`로 치환.
   - 동일 파일이 있으면 `_2`, `_3` 식으로 접미사.

5. **결과 요약**
   - 저장한 파일 경로를 사용자에게 알리고, 레시피의 요약(요리명, 예상 조리시간, 사용 재료 목록)을 응답에 표시.

## 마크다운 출력 포맷

저장되는 파일 구조:

```markdown
# {요리명}

> 1인분 · 조리시간 {N}분 · 자취생 난이도

## 사용한 냉장고 재료
- 계란 2개
- 김치 한 줌
- ...

## 추가로 필요한 재료 (있으면)
- 간장 1큰술
- ...

## 조리 순서
1. ...
2. ...
3. ...

## 팁
- ...

---
생성일: {YYYY-MM-DD HH:mm}
사용한 재료 파일: ingredients/egg.json, ingredients/kimchi.json, ...
```

## 주의사항

- 실제로 읽어들인 재료 이름과 수량을 **근거로만** 레시피를 제안하세요. 없는 재료를 "있는 척"하지 마세요.
- 재료의 `category`가 `냉동`이면 해동 시간을 고려해 15분 안에 조리 가능한지 판단하세요.
- 사용자가 특정 요리 타입을 원하면(`/recipe 국물요리`, `/recipe 간식`) 그 제약을 우선 반영합니다.
- `ingredients/` 안의 파일을 **수정·삭제하지 마세요** — 이 스킬은 읽기 전용입니다. 재료 추가/삭제는 사용자가 파일을 직접 관리합니다.
