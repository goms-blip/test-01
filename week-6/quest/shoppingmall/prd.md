
## 📝 [PRD] 쇼핑몰 서비스 완성 (이미지 + 결제 + 마이페이지)

## 1. 프로젝트 개요

* 목적: 기존 5주차 쇼핑몰에 이미지 업로드, 실제 결제 시스템, 주문 내역 관리 기능을 추가하여 실서비스 수준의 완성도 구현.
* 핵심 목표:
* 외부 스토리지(ImageKit) 연동을 통한 이미지 관리.
   * 토스페이먼츠 위젯을 활용한 안전한 결제 플로우 구축.
   * 사용자별 주문 내역 조회(마이페이지) 및 데이터 무결성 확보.

## 2. 주요 기능 요구사항

## Part 1: 관리자 상품 이미지 업로드

* ImageKit 연동: 상품 이미지를 외부 스토리지에 업로드하고 반환된 URL을 DB(products 테이블)에 저장.
* UI/UX: 상품 등록 폼에서 드래그앤드롭 또는 파일 선택 기능 제공.
* 검증: 상품 목록 페이지에서 저장된 이미지가 정상적으로 출력되는지 확인.

## Part 2: 결제 시스템 (Toss Payments)

* 결제 위젯: 장바구니에서 결제 페이지로 이동 시 토스페이먼츠 결제 위젯 렌더링.
* 보안 처리:
* Client Key는 프론트엔드에서 사용.
   * Secret Key는 반드시 서버 환경변수로 관리하여 승인 API 호출 시 사용 (노출 금지).
* 데이터 처리: 결제 승인 완료 후 orders 및 order_items 테이블에 주문 정보 저장.

## Part 3: 마이페이지 (주문 관리)

* 조회 기능: 로그인한 사용자의 user_id를 기준으로 주문 내역 필터링.
* 표시 항목: 상품명, 총 결제 금액, 주문 일시, 주문 번호(ID).
* 권한 제어: 타인의 주문 내역에 접근할 수 없도록 서버 측 권한 검증.

## Part 4: 배포 및 환경 설정

* Vercel 배포: 전체 서비스를 Vercel을 통해 호스팅.
* 환경변수 관리: TOSS_SECRET_KEY, IMAGEKIT_PRIVATE_KEY 등 민감 정보를 Vercel 대시보드에 등록.

## 3. 데이터베이스 구조 (참고)

* products: id, name, price, image_url, description
* orders: id, user_id, total_price, paid_at, status
* order_items: id, order_id, product_id, quantity, price

## 4. 핵심 로직 흐름

   1. 상품 등록: 이미지 선택 → ImageKit 업로드 → URL 획득 → DB Insert.
   2. 결제: 결제 요청 → 토스 인증 → 서버 승인 API 호출 → DB Order 저장 → 결과 페이지 리다이렉트.
   3. 마이페이지: 세션 확인 → user_id 쿼리 → 주문 목록 렌더링.

## 5. 최종 제출 체크리스트

* Vercel 배포 URL
* GitHub 저장소 링크
* 스크린샷 (상품 목록, 장바구니, 결제창, 마이페이지)
* 테스트 결제 성공 화면 캡처
