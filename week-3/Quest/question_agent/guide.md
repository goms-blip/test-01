# 🤖 프로젝트 기획안: 나만의 AI 학습 조교 에이전트 (My AI Tutor)

이 프로젝트는 수업에서 배운 **텍스트(이론)와 코드(실습)** 데이터를 통합하여, 맥락에 맞는 정확한 답변을 제공하는 개인형 학습 지원 에이전트입니다.

---

## 1. 🎯 프로젝트 개요
- **목적:** 수업 자료와 코드를 학습한 AI를 통해 궁금증을 즉시 해결하고, 이전 대화 맥락을 유지하며 심화 학습 지원
- **핵심 기능:** 
  - 텍스트/코드 컨텍스트 병합 참조
  - LangChain 또는 OpenAI Memory를 활용한 대화 흐름 유지
  - 수업 실습 코드 기반의 예제 재구성

## 2. 🛠 기술 스택 및 구조
- **언어:** Python
- **프레임워크:** LangChain (LLM 오케스트레이션)
- **LLM:** OpenAI GPT-4o (또는 GPT-3.5-turbo)
- **메모리:** `ConversationBufferMemory` 또는 `PostgreSQL` 기반 대화 기록 저장
- **데이터 로더:** `DirectoryLoader`, `TextLoader` (강의 노트 및 `.py`, `.js` 코드 파일 로드용)

## 3. 📚 데이터 구성 (Context)
에이전트는 다음 두 가지 데이터를 기반으로 지식을 구성합니다.
1. **Text Context (`/docs/*.md`):** 강의 슬라이드 요약, 주요 개념 정리, 수업 일정 등
2. **Code Context (`/src/*.py`):** 수업 중 작성한 실습 코드, API 라우팅 예시, DB 스키마 등

## 4. 🧠 에이전트 구현 전략
- **RAG (Retrieval-Augmented Generation):** 수업 자료가 많을 경우 Vector DB(Chroma 등)에 저장하여 관련 내용을 검색해 답변.
- **System Prompt:** 
  > "너는 [강의명] 수업의 전문 조교 AI야. 제공된 수업 노트와 코드를 바탕으로만 답변하며, 모르는 내용은 추측하지 말고 수업 자료를 확인하라고 안내해줘."
- **Memory:** `chat_history` 변수를 프롬프트에 포함하여 "아까 말한 내용"이 무엇인지 인지하도록 설정.

## 5. 🏃‍♂️ 시나리오 워크플로우
1. **질문 1 (개념):** "지난 수업에서 배운 API 라우팅 방식이 뭐였지?"
   - AI: `docs/lecture_01.md`를 참조하여 라우팅 개념 설명
2. **질문 2 (코드):** "그때 작성한 코드 예제 보여줘."
   - AI: `src/main.py` 파일 내의 `@app.get('/')` 코드 블록을 추출하여 제시
3. **질문 3 (연결):** "그 방식이랑 아까 말한 보안 설정을 연결해서 설명해줘."
   - AI: 메모리에 저장된 '라우팅' 내용과 '보안' 개념을 결합하여 답변
